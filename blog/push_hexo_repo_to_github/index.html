Hexo将Github Page部署到Github时可能发生的一点问题
2024年4月1日
•
1 min read
Categories: 技术 / Hexo相关
Tags: Github / Hexo

很奇怪的事情。昨天下午我从自习室回宿舍之前把昨天的刷题日记写完了，推送到Github，然后Hexo给我抛了一个错误

Could not read from remote repository

多次部署都抛了这个错误

我寻思着前一天部署也没出问题啊，我也没有改过config文件

只能去Google找解决方案了

Google给了两个解决方案：一个是修改远程仓库的域名：把域名修改成Git clone的域名，另一个是在本机上重新搞一个ssh密钥，然后上传到Github。我寻思着我这个ssh密钥还是几周前刚做的，不可能那么快就失效了，于是选择了第一个方法

然后，很成功地推送上去了

然后，我今天登本站，居然没显示新内容！

又有新问题了！我多次尝试修改文件，重新构建静态文件，然后推送到Github，都没法显示新内容，但是Hexo服务端在本机运行的时候我是能看到新内容的。也就是说Hexo没问题，是我远程仓库的域名设置有问题？

我又把域名修改回了原来的域名。很奇怪，今天就没有出现 “Could not read from remote repository” 这样的报错了，成功把新页面推送到了Github上

很奇怪的事故。到现在我都不清楚到底是哪里出现了问题，是昨天网络环境有问题，导致没法读取Github上的远程仓库吗？我不清楚
